Alternatives to Databricks Auto Loader
========================================

Current Auto Loader Pipeline uses `cloudFiles` format for incremental streaming ingestion.


1. COPY INTO (SQL / Batch Incremental)
---------------------------------------
The closest alternative — idempotent, tracks loaded files, runs as a batch SQL command.

    COPY INTO bronze_table
    FROM 's3://your-bucket/your-path'
    FILEFORMAT = JSON
    COPY_OPTIONS ('mergeSchema' = 'true')

- Batch (not streaming)
- Simpler to run on a schedule
- Good for lower-volume, scheduled ingestion


2. Standard Structured Streaming (no cloudFiles)
--------------------------------------------------
Uses PySpark's built-in file stream reader — no Auto Loader-specific features.

    df = spark.readStream.format("json").load("s3://your-bucket/your-path")
    df.writeStream.format("delta") \
        .option("checkpointLocation", "/tmp/checkpoint") \
        .outputMode("append") \
        .table("bronze_table")

- Streaming, but no schema evolution or file tracking from Auto Loader
- Less scalable at high file volumes


3. Delta Live Tables (DLT)
---------------------------
Databricks declarative pipeline framework — wraps Auto Loader in a managed pipeline.

    import dlt
    from pyspark.sql.functions import col

    @dlt.table
    def bronze_table():
        return (
            spark.readStream.format("cloudFiles")
            .option("cloudFiles.format", "json")
            .option("cloudFiles.schemaLocation", "/Volumes/schema")
            .load("s3://your-bucket/your-path")
        )

- Managed orchestration, lineage, data quality checks
- Recommended for production pipelines


4. PySpark Batch (spark.read)
------------------------------
Full batch — reads all files every run. No incremental state.

    df = spark.read.format("json").load("s3://your-bucket/your-path")
    df.write.format("delta").mode("append").saveAsTable("bronze_table")

- Simple, but re-reads everything — not efficient for large/growing datasets


Summary
--------
Approach                          | Mode      | Incremental       | Complexity
----------------------------------|-----------|-------------------|------------
Auto Loader (cloudFiles)          | Streaming | Yes (automatic)   | Medium
COPY INTO                         | Batch     | Yes (file tracking)| Low
Standard Structured Streaming     | Streaming | Yes (checkpoint)  | Medium
Delta Live Tables                 | Both      | Yes               | High (managed)
PySpark Batch                     | Batch     | No                | Low

Most common alternative for scheduled batch pipelines: COPY INTO
For production streaming pipelines: Delta Live Tables (DLT)
